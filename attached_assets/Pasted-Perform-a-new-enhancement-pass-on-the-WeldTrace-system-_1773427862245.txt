Perform a new enhancement pass on the WeldTrace system.

Goal:

1. reduce database storage using curve compression
2. upgrade the welding report to a professional engineering report
3. add a verification payload for QR-based weld validation

Do not modify the welding calculation engine.

---

# 1 — Compress stored welding curve

Currently the weld curve is stored as JSON:

traceCurveJson TEXT

Large welds may reach several thousand samples.

Implement gzip compression before storing.

Create helper:

services/welding_trace/curve_compression.dart

Functions:

Uint8List compressCurve(String json)

String decompressCurve(Uint8List data)

Implementation:

use dart:convert
use gzip encoder

Example:

final bytes = utf8.encode(json)
final compressed = gzip.encode(bytes)

Return Uint8List.

---

# 2 — Update database schema

Replace column:

traceCurveJson TEXT

with

traceCurveCompressed BLOB

Migration:

if upgrading from schema < 6:

add new column traceCurveCompressed BLOB

Keep traceCurveJson for backward compatibility.

---

# 3 — Update DAO saveTraceData()

Before saving curve:

curveJson = jsonEncode(curve)

compressedCurve =
CurveCompression.compressCurve(curveJson)

Save compressedCurve into traceCurveCompressed.

Keep original JSON column optional.

# 4 — Update weld load logic

# 6 — Add weld verification payload

The QR code must contain a verification JSON payload.

Structure:

{
"app": "WeldTrace",
"version": 1,
"signature": weldSignature,
"machine": machineId,
"diameter": pipeDiameter,
"material": material,
"sdr": sdr,
"timestamp": timestamp
}

Encode payload as JSON string.

Generate QR from that JSON.

---

# 7 — Add weld verification helper

Create file:

services/welding_trace/weld_verifier.dart

WeldVerifier

bool verifySignature({
required String signature,
required String machineId,
required double diameter,
required String material,
required String sdr,
required List<WeldTracePoint> curve,
